# PC 端前端与后端匹配分析报告

## 总体情况

**结论：前端所有页面都使用假数据，字段结构与后端完全不匹配，需要全面重构。**

---

## 1. 工单管理（WorkOrder.vue）

### 前端字段
```javascript
{
  id, clinicName, customerName, phone, address,
  senderAddress, returnAddress, productModel, itemsList,
  fault, status, isHandled, submitTime,
  logisticsCompany, logisticsNo, returnCompany, returnNo,
  needInvoice, invoiceTitle, taxId, invoiceStatus, images
}
```

### 后端字段（cicada_orders + cicada_order_items）
```javascript
// cicada_orders
{
  _id, order_no, user_id, status,
  ship_out_info: { name, phone, region, detail, logistics_company, logistics_no },
  ship_back_info: { name, phone, region, detail },
  engineer_id, total_price, timeline, create_time, update_time
}

// cicada_order_items
{
  _id, order_id, product_name, product_model, sn,
  buy_date, fault_desc, media_urls, fix_solution
}
```

### 主要不匹配点
1. **状态值不匹配**
   - 前端：'待处理', '维修中', '已发货'
   - 后端：'pending', 'received', 'inspecting', 'fixing', 'shipped', 'completed', 'cancelled'

2. **物流信息结构不匹配**
   - 前端：平铺字段（senderAddress, logisticsCompany 等）
   - 后端：嵌套对象（ship_out_info, ship_back_info）

3. **缺少后端字段**
   - engineer_id（分配工程师）
   - timeline（时间线）
   - total_price（总价）

4. **多余的前端字段**
   - needInvoice, invoiceTitle, taxId, invoiceStatus（后端数据库没有发票相关字段）
   - isHandled（后端没有此字段）

## 2. 故障库管理（FaultDB.vue）

### 前端字段
```javascript
// 分类
{ id, name }

// 故障
{ id, catId, name, desc, solution, createdAt }
```

### 后端字段
```javascript
// cicada_product_categories
{ _id, category_name, status, sort }

// cicada_fault_kb
{
  _id, category_id, fault_name,
  related_questions: [],
  check_steps: [],
  fix_solutions: [],
  is_recommend_repair
}
```

### 主要不匹配点
1. **字段名不匹配**
   - 前端：name → 后端：category_name / fault_name
   - 前端：catId → 后端：category_id

2. **缺少后端字段**
   - status（分类状态）
   - sort（排序）
   - related_questions（相关问题数组）
   - check_steps（检查步骤数组）
   - fix_solutions（修复方案数组）
   - is_recommend_repair（是否推荐返修）

3. **字段类型不匹配**
   - 前端：solution 是字符串
   - 后端：fix_solutions 是数组

## 3. 反馈管理（Feedback.vue）

### 前端字段
```javascript
{
  id, submitTime, type, customerName, phone,
  content, status
}
```

### 后端字段（cicada_feedbacks）
```javascript
{
  _id, user_id, type, content, images,
  contact_type, contact_value, rel_order_no,
  status, create_time
}
```

### 主要不匹配点
1. **状态值不匹配**
   - 前端：'未读', '已处理'
   - 后端：'待处理', '已处理'

2. **类型值不匹配**
   - 前端：'投诉', '建议'
   - 后端：'投诉', '建议'（这个匹配）

3. **缺少后端字段**
   - user_id（用户ID）
   - images（图片数组）
   - contact_type, contact_value（联系方式）
   - rel_order_no（关联工单号）

4. **字段名不匹配**
   - 前端：submitTime → 后端：create_time
   - 前端：customerName, phone → 后端：需要通过 user_id 关联 cicada_users 表获取

## 4. 用户管理（Users.vue）

### 前端字段
```javascript
{
  id, name, phone, role, active
}
```

### 后端字段（cicada_users）
```javascript
{
  _id, openid, username, password_hash, password_salt,
  token, token_expire, role, phone, name, nickname,
  avatar, disabled, last_login, create_time
}
```

### 主要不匹配点
1. **角色值不匹配**
   - 前端：'管理员', '调度员', '工程师'
   - 后端：'user', 'admin', 'engineer'（没有'调度员'角色）

2. **状态字段不匹配**
   - 前端：active（布尔值）
   - 后端：disabled（布尔值，含义相反）

3. **缺少后端字段**
   - username（登录用户名）
   - password（密码，新增时需要）
