## 你需要完成的配置步骤

### 第 1 步：在 uniCloud 控制台开启云函数 URL 化

1. 打开浏览器，访问 [uniCloud 控制台](https://unicloud.dcloud.net.cn/)
2. 登录你的账号
3. 选择你的云服务空间
4. 点击左侧菜单"云函数/云对象"
5. 依次为以下 3 个云函数开启 URL 化：

   **cicada-admin-sys**（管理员登录、员工管理）
   - 点击函数名进入详情
   - 找到"云函数URL化"开关，点击开启
   - 复制生成的 URL（类似：`https://fc-mp-xxxxx.next.bspapp.com/http/cicada-admin-sys`）

   **cicada-admin-kb**（知识库管理）
   - 同样操作，复制 URL

   **cicada-admin-order**（工单管理）
   - 同样操作，复制 URL

### 第 2 步：更新 API 配置

复制 `.env.example` 为 `.env.local`：

```powershell
Copy-Item .env.example .env.local
```

推荐配置统一云空间基地址：

```bash
VITE_UNICLOUD_BASE_URL=https://你的云函数URL域名
```

如果每个云函数 URL 独立，也可以分别配置：

```bash
VITE_ADMIN_SYS_URL=https://你的 cicada-admin-sys URL
VITE_ADMIN_KB_URL=https://你的 cicada-admin-kb URL
VITE_ADMIN_ORDER_URL=https://你的 cicada-admin-order URL
```

如果已为公共云对象开启 URL 化，也可以额外配置：

```bash
VITE_CLIENT_PUBLIC_URL=https://你的 cicada-client-public URL
```

未配置 `VITE_CLIENT_PUBLIC_URL` 时，`npm run check:urls` 会使用 `cicada-admin-order/getSubscriptionConfig` 只读端点检查订阅配置通道。配置完成后运行 `npm run check:urls`，确认后台 URL 已命中最新部署的云函数。

### 第 3 步：创建双核心管理员账号

由于 PC 端需要用管理员账号登录，上线交付时需要在数据库中预置两个最高权限账号，避免主管理员遗忘密码后系统死锁。

**方式 1：通过 uniCloud 控制台直接插入数据**

1. 进入 uniCloud 控制台 → 云数据库 → `cicada_users` 集合
2. 点击"添加记录"，分别插入 `admin` 与 `admin_root` 两条数据：

```json
{
  "openid": "admin_openid",
  "username": "admin",
  "password": "初始明文密码",
  "password_hash": "",
  "password_salt": "",
  "role": "admin",
  "name": "日常主管理员",
  "disabled": false,
  "create_time": 当前时间戳
}
```

```json
{
  "openid": "admin_root_openid",
  "username": "admin_root",
  "password": "高强度复杂密码",
  "password_hash": "",
  "password_salt": "",
  "role": "admin",
  "name": "紧急救援账号",
  "disabled": false,
  "create_time": 当前时间戳
}
```

首次登录成功后，`cicada-admin-sys` 会自动把明文 `password` 迁移为 `password_hash + password_salt` 并清空明文密码。`admin_root` 仅作底层救援账号，密码需物理隔离保存，日常严禁使用；系统也会拦截对 `admin_root` 的一键重置密码操作。

**方式 2：使用维护云函数（推荐）**

小程序项目中已有 `cicada-maintenance` 云函数，可以自动将明文密码转换为哈希。

### 账号安全与密码管理

- 右上角【修改密码】：所有后台人员可自助修改自己的登录密码，需校验原密码。修改成功后会清空当前 Token 并强制回到登录页。
- 用户管理【重置密码】：仅主管理员可将员工账号密码重置为系统默认密码 `123456`，重置后需线下通知用户立即自行修改。
- 死锁救援闭环：若日常主账号 `admin` 遗忘密码，可使用 `admin_root` 登录后台，在用户管理中重置 `admin` 密码。
- 架构说明：仅需新增 2 个前端小弹窗和 2 个管理端云对象接口。

## 常见问题

**Q: 登录时提示"网络错误"？**  
A: 检查 `src/config/api.js` 中的 URL 是否正确，确保云函数已开启 URL 化。

**Q: 登录时提示"账号或密码错误"？**  
A: 确保数据库中已创建管理员账号，且密码已正确哈希。

**Q: 如何创建第一个管理员账号？**  
A: 可以临时修改 `cicada-admin-sys` 云函数，添加一个初始化接口来创建管理员。
