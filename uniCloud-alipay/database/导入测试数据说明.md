# 测试数据导入说明

## 数据文件列表

1. `test-data.json` - 用户和产品分类
2. `test-data-fault-kb.json` - 故障知识库
3. `test-data-addresses.json` - 用户地址
4. `test-data-orders.json` - 订单
5. `test-data-order-items.json` - 订单项目

## 导入步骤

### 方式一：通过 uniCloud Web 控制台导入

1. 登录 uniCloud Web 控制台
2. 选择对应的云空间
3. 进入"云数据库"
4. 对每个集合（表）进行导入：
   - 点击集合名称
   - 点击"导入"按钮
   - 选择对应的 JSON 文件
   - 确认导入

### 方式二：通过 HBuilderX 导入

1. 在 HBuilderX 中右键点击 `uniCloud-alipay`
2. 选择"云服务空间初始化向导"
3. 选择对应的云空间
4. 在云数据库面板中，右键点击集合
5. 选择"导入数据"
6. 选择对应的 JSON 文件

## 注意事项

1. **导入顺序**：建议按以下顺序导入
   - cicada_users
   - cicada_product_categories
   - cicada_fault_kb
   - cicada_addresses
   - cicada_orders
   - cicada_order_items

2. **_id 字段**：如果导入时提示 _id 冲突，可以删除 JSON 中的 _id 字段，让数据库自动生成

3. **时间戳**：测试数据中的时间戳可能需要更新为当前时间

4. **索引创建**：导入数据后，请参考 `INDEXES.md` 创建必要的数据库索引
