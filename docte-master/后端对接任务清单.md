# 后端对接任务清单

这份文档用于交接给后端同学，目标是把当前小程序前端真正依赖的接口、字段和上线检查项说清楚。更完整的项目说明见 [README.md](./README.md)。

## 1. 当前前端状态

- 前端框架：`uni-app + Vue 3`
- 运行端：微信小程序
- 主请求封装：[`utils/request.js`](./utils/request.js)
- 主业务接口：[`api/content.js`](./api/content.js)
- 登录页：[`pages/login/index.vue`](./pages/login/index.vue)
- 首页主入口：[`pages/index/index.vue`](./pages/index/index.vue)

- 收货地址页：[`pages/address/index.vue`](./pages/address/index.vue)

当前状态有两点要先说明：

- 首页主流程已经切到 HTTP 接口。
- 登录页目前仍保留 `uniCloud.importObject('cicada-client-user')` 的云对象调用。

## 2. 请求与返回约定

### 2.1 请求

- 接口基地址优先读取 `.env.local` 里的 `VITE_API_BASE_URL`
- 未配置时默认使用 `https://api.cisco-d.com/api/v1`
- 请求封装会自动带上 `Authorization: Bearer {token}`

### 2.2 返回

建议统一返回：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

约定：

- `code = 0` 表示成功
- `401 / 1004 / 100401` 视为登录失效
- 登录失效后前端会清理本地 `token` 和 `userInfo`

## 5. 关键数据约定

### 5.1 登录返回

建议 `userInfo` 至少包含：

- `id`
- `phone`
- `nickname`
- `unit`
- `avatar`

### 5.2 工单状态

前端会识别这些常见状态：

- `pending` / `已提交`
- `checking` / `检测中`
- `quote_pending` / `待报价`
- `waiting_confirm` / `待确认`
- `fixing` / `维修中`
- `shipped` / `已发货`
- `completed` / `已完成`

建议后端同时返回 `status` 和 `statusText`，避免前端再做大量映射。

### 5.3 包裹查询

建议至少返回：

- `trackingNo`
- `company`
- `status`
- `orderId`
- `timeline`

### 5.4 上传结果

上传接口建议返回：

- `url`
- 或 `fileId`

前端会把上传结果继续带入报修、回寄、发票等流程。

## 6. 数据库与索引

上线前请先确认 [uniCloud-alipay/database/INDEXES.md](./uniCloud-alipay/database/INDEXES.md) 里的索引已经创建，重点是：

- `cicada_orders.order_no` 必须唯一
- `cicada_orders.user_id + create_time`
- `cicada_orders.status + create_time`
- `cicada_order_items.order_id`
- `cicada_users.token / openid / username / role`

## 7. 上线前检查

- 确认 `VITE_API_BASE_URL` 指向正确环境
- 确认 uniCloud 环境变量已配置 `WX_APPID` 和 `WX_SECRET`
- 确认登录方式继续保留云对象，还是统一改成 HTTP
- 确认图片 CDN 地址已替换
- 确认接口返回格式统一为 `code / message / data`
- 确认微信开发者工具能正常编译和预览

## 8. 交接顺序建议

1. 先打通登录
2. 再打通报修提交、列表和详情
3. 再补包裹查询、政策和客服
4. 最后补地址、投诉、发票、产品和管理端能力
